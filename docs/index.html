Overview

Dotenv C++ is a lightweight library that allows you to load environment variables from .env files into your application. This is particularly useful for managing configuration across different environments without hardcoding sensitive information.

Installation

Download

Download the latest release from the GitHub releases page.

Files Included

  • dotenv.dll - The shared library
  • libdotenv.a - Import library for linking
  • dotenv.hpp - C++ header file

Quick Setup

  1. Copy dotenv.dll and libdotenv.a to your project's library directory
  2. Copy dotenv.hpp to your project's include directory
  3. Link against the library in your build configuration

Usage

1. Create a .env file

# .env file
DATABASE_URL=postgresql://localhost/mydb
API_KEY="your-api-key-here"
PORT=3000
DEBUG=true

2. C++ API

The simplest way to use the library:

#include "dotenv.hpp"

int main() {
    // Load environment variables from .env
    Dotenv::load(".env");
    
    // Access variables using standard methods
    const char* dbUrl = std::getenv("DATABASE_URL");
    const char* apiKey = std::getenv("API_KEY");
    
    // Use your environment variables
    std::cout << "Database: " << dbUrl << std::endl;
    
    return 0;
}

3. C API

For C compatibility:

#include "dotenv.hpp"
#include <stdlib.h>
#include <stdio.h>

int main() {
    // Load environment variables
    CallDotenvLoad(".env");
    
    // Access variables
    const char* port = getenv("PORT");
    printf("Port: %s\n", port);
    
    return 0;
}

4. Dynamic Runtime Loading (Windows)

Load the DLL at runtime without static linking:

#include <windows.h>
#include <iostream>

typedef void (*CallDotenvLoadFunc)(const char*);

int main() {
    // Load the DLL at runtime
    HMODULE hDll = LoadLibrary("dotenv.dll");
    if (!hDll) {
        std::cerr << "Failed to load dotenv.dll" << std::endl;
        return 1;
    }

    // Get the function pointer
    CallDotenvLoadFunc loadFunc = 
        (CallDotenvLoadFunc)GetProcAddress(hDll, "CallDotenvLoad");
    
    if (!loadFunc) {
        std::cerr << "Failed to find CallDotenvLoad" << std::endl;
        FreeLibrary(hDll);
        return 1;
    }

    // Call the function
    loadFunc(".env");
    
    // Use environment variables
    std::cout << "API Key: " << std::getenv("API_KEY") << std::endl;
    
    // Clean up
    FreeLibrary(hDll);
    return 0;
}

.env File Format

Supported Features:
  • Comments start with #
  • Key-value pairs: KEY=value
  • Quoted values: KEY="value with spaces"
  • Empty lines are ignored
# Database Configuration
DATABASE_URL=postgresql://localhost:5432/mydb
DB_USER=admin
DB_PASSWORD="secure_password_123"

# API Settings
API_KEY="sk-1234567890abcdef"
API_ENDPOINT=https://api.example.com

# Application Settings
PORT=3000
DEBUG=true
LOG_LEVEL=info

API Reference

C++ API

Dotenv::load()

static void Dotenv::load(const std::string& filename = ".env");

Description: Loads environment variables from the specified file.

Parameter Type Description
filename const std::string& Path to the .env file (default: ".env")

C API

CallDotenvLoad()

void CallDotenvLoad(const char* filename_c_str);

Description: C-compatible function to load environment variables.

Parameter Type Description
filename_c_str const char* Path to the .env file (NULL uses ".env")

Return Values

Both functions return void. If the file cannot be opened, an error message is printed to stderr, but the program continues execution.

Building from Source

Requirements

  • g++ compiler with C++11 support
  • GNU Make
  • Windows SDK (for code signing on Windows)

Build Commands

Build the Library

make bin/dotenv.dll

Clean Build Artifacts

make clean

Compiler Flags

The library is built with the following flags:

  • -std=c++11 - C++11 standard
  • -O2 - Optimization level 2
  • -Wall -Wextra - All warnings enabled
  • -DBUILDING_DOTENV_DLL - Export symbols for DLL
  • -static-libgcc -static-libstdc++ - Static linking of runtime

Output Files

  • bin/dotenv.dll - Shared library
  • bin/libdotenv.a - Import library
  • bin/dotenv.o - Object file

Code Signing (Windows)

For production deployments, you can sign the DLL to verify its authenticity.

Create Test Certificate

For testing purposes, create a self-signed certificate:

powershell -ExecutionPolicy Bypass -File create-cert.ps1
‚ö†Ô∏è Warning: Self-signed certificates will trigger security warnings. For production use, obtain a certificate from a trusted Certificate Authority (CA).

Sign the DLL

make sign

This uses the default certificate (certificate.pfx) with password dotenv123.

Custom Certificate

To use a different certificate:

make sign CERT_FILE=path/to/cert.pfx CERT_PASSWORD=yourpassword

Examples

Web Server Configuration

#include "dotenv.hpp"
#include <iostream>
#include <cstdlib>

int main() {
    Dotenv::load(".env");
    
    const char* host = std::getenv("SERVER_HOST");
    const char* port = std::getenv("SERVER_PORT");
    const char* debug = std::getenv("DEBUG");
    
    std::cout << "Starting server on " 
              << (host ? host : "localhost") 
              << ":" 
              << (port ? port : "8080") 
              << std::endl;
    
    if (debug && std::string(debug) == "true") {
        std::cout << "Debug mode enabled" << std::endl;
    }
    
    return 0;
}

            return;
        }

    }

    return 0;
}

Troubleshooting

DLL Not Found

If you get an error that the DLL cannot be loaded:

  • Ensure dotenv.dll is in the same directory as your executable
  • Add the DLL directory to your system PATH
  • Use an absolute path when calling LoadLibrary()

Linking Errors

If you get linking errors:

  • Make sure you're linking against libdotenv.a
  • Verify the library is in your linker's search path
  • Check that BUILDING_DOTENV_DLL is NOT defined when building your application

Environment Variables Not Loading

  • Verify the .env file path is correct
  • Check file permissions
  • Ensure the file format is correct (key=value pairs)
  • Check stderr for error messages
